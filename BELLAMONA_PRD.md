## 1. 제품 개요

### 1.1 한 줄 정의
벨라모나(Bellamona)는 PCOS(다낭성난소증후군) 및 호르몬 밸런스를 관리하려는 여성을 위해, **혈당(인슐린) · 운동(성장호르몬) · 수면(멜라토닌) · 감정(옥시토신)** 4개 축의 데이터를 통합 기록하고, 3일 주기로 AI가 상관관계 리포트를 발행하는 모바일 앱이다.

### 1.2 현재 상태 (As-Is)
· bellamona.net — Vercel 배포, 단일 HTML/JS 파일 웹앱
· Google 로그인 UI 존재(실질 세션/서버 인증 미비), 데이터는 클라이언트 로컬 저장 중심
· 구현된 UI: 온보딩(성별/목표/생리주기), TODAY(칼로리 NET, 4대 호르몬 루틴 달성률), DASHBOARD(월간 저속노화 점수, 달력, 생리기/배란기), 저당 레시피, 운동 칼로리 기록, 일기, 친구 연동, 치팅데이, 구독 페이월(월 9,900원)
· 미구현: 실제 백엔드/DB, 실제 AI 리포트 생성, 혈당 예측, CGM 연동, 결제, 앱 패키징

### 1.3 목표 상태 (To-Be, v1.0 앱 런칭)
· iOS/Android 스토어에 출시된 앱 (웹 코드베이스 기반 하이브리드)
· 서버 DB에 안전하게 저장되는 사용자 데이터 (건강정보 = 민감정보 취급)
· 실제 작동하는 3일 주기 AI 통합 리포트 (일기 키워드 × 혈당 × 운동 × 수면 상관 분석)
· 식사 기록 기반 개인화 혈당 반응 예측 v1
· CGM(연속혈당측정기) 데이터 유입 경로 확보 (HealthKit/Health Connect 경유)
· 인앱 구독 결제 (월 9,900원)

### 1.4 호르몬 4축 재정의 (중요 — 기존 사이트와 차이)
기존 사이트의 4축은 인슐린/성장호르몬/코르티솔/옥시토신이었다. v1.0에서는 **코르티솔 → 멜라토닌(수면)** 으로 교체한다.
· 🩺 인슐린 = 식단·혈당 (음식 기록, 혈당 기록, CGM)
· 💪 성장호르몬 = 운동 (근력/HIIT/유산소, 소모 칼로리, 수면 중 분비와의 연결 설명)
· 🌙 멜라토닌 = 수면 (취침/기상 시각, 수면 시간, 수면 질 자가평가)
· 💗 옥시토신 = 감정·유대 (일기, 감정 태그, 친구 연동 활동)
※ 기존 사용자의 코르티솔 루틴 데이터가 있다면 멜라토닌 축으로 마이그레이션하되 원본은 보존한다.

## 2. 타깃 사용자 & 문제 정의

### 2.1 페르소나
· P1 "혜림" (핵심): 20대 후반~30대 여성, PCOS 진단 또는 의심, 인슐린 저항성 관리 필요, 체중 감량과 생리주기 안정이 목표. CGM(리브레 등)을 자비로 써본 경험 있거나 관심 있음.
· P2 "저속노화族": 질병은 없지만 혈당 스파이크 관리·저속노화 트렌드를 따르는 25~45세 여성.
· P3 "임신 준비": 배란·인슐린 감수성 관리가 임신 준비와 직결되는 사용자.

### 2.2 해결하는 문제
· 혈당앱 따로, 운동앱 따로, 감정일기 따로 — 통합 해석이 없다.
· "내가 스트레스 받으면 → 야식을 먹고 → 혈당이 튀고 → 잠을 못 자는" 악순환의 **개인 패턴**을 아무도 보여주지 않는다.
· CGM을 사도 데이터 해석을 못 해서 2주 쓰고 서랍행이 된다.

### 2.3 핵심 가치 제안
"3일마다 받는 나만의 호르몬 리포트: 내 감정 단어가 내 혈당 그래프와 어떻게 연결되는지 보여준다."

## 3. 기능 요구사항 (Functional Requirements)

### F-01. 계정 & 온보딩 — P0
· Google/Apple 소셜 로그인 (Apple은 iOS 심사 필수 요건: 3rd party 로그인 제공 시 Sign in with Apple 필수)
· 온보딩: 닉네임/아바타 → 목표(PCOS 관리·임신 준비·저속노화·체지방 감량) → 신체정보(키/체중/목표체중/기간) → 생리주기 입력 → 민감정보 처리 동의(필수 별도 동의) → 알림 권한
· 만 14세 미만 가입 차단 (생년 입력 + 자체 검증)

### F-02. 식단·혈당 기록 (인슐린 축) — P0
· 음식 기록: 텍스트 검색(식약처 식품영양성분 DB 기반) + 사진 촬영 → AI 음식 인식(품목/양 추정 → 사용자 확인 후 저장)
· 기록 항목: 음식명, 양, 칼로리, 탄/단/지, GI 추정치, 식사 시각
· 혈당 수동 기록: 시각 + 수치(mg/dL) + 식전/식후 태그
· CGM 데이터: HealthKit(iOS)/Health Connect(Android)에서 혈당 샘플 읽기 권한 요청 → 자동 동기화 (리브레·덱스콤이 헬스 플랫폼에 기록한 데이터 활용)
· 식후 혈당 곡선 시각화: 식사 마커 + 혈당 라인차트

### F-03. 혈당 반응 예측 — P0 (v1은 개인화 규칙 기반, v2에서 모델 고도화)
· 사용자가 음식을 기록(또는 기록 전 "이거 먹으면?" 시뮬레이션)하면 예상 혈당 반응을 3단계(낮음/중간/높음 스파이크) + 예상 피크 범위로 표시
· v1 로직: (a) 음식의 탄수화물량·GI 기반 베이스라인 → (b) 해당 사용자의 과거 유사 식사 실측 혈당 반응으로 보정 → (c) 시간대/생리주기 phase(황체기 인슐린 감수성 저하) 보정
· 예측 결과에는 반드시 "참고용 추정치이며 의료적 판단이 아닙니다" 고지 표시
· 유사 식사 데이터 3건 미만이면 "데이터가 더 쌓이면 정확해져요" 상태로 베이스라인만 표시

### F-04. 운동 기록 (성장호르몬 축) — P0
· 기존 UI 유지: 근력/HIIT/공복 유산소/걷기, 부위 선택, 소모 칼로리
· HealthKit/Health Connect에서 걸음수·운동 세션 자동 가져오기 — P1

### F-05. 수면 기록 (멜라토닌 축) — P0
· 취침/기상 시각 입력 → 수면시간 자동 계산, 수면 질 1~5 자가평가
· HealthKit/Health Connect 수면 데이터 자동 동기화 — P1

### F-06. 감정 일기 (옥시토신 축) — P0
· 하루 1회 이상 자유 텍스트 일기 + 감정 태그(선택)
· 저장 시 서버에서 키워드·감정 분석(경량 AI) 수행, 결과를 diary_analysis에 저장
· 분석 산출물: 감정 점수(-1~+1), 상위 감정 카테고리, 키워드 목록(음식 갈망/스트레스원/관계 관련 단어 분류)

### F-07. AI 통합 리포트 (핵심 차별화) — P0
· 주기: 최근 3일(72시간)의 데이터가 최소 기준(일기 1건 + 식사 3건 + 혈당 데이터 존재)을 충족하면 생성 가능. 구독자는 3일마다 자동 생성 + 푸시.
· 리포트 구성:
  1) 3일 요약 스코어 (4축 각각 0~100)
  2) 감정 키워드 클라우드 & 감정 곡선
  3) 혈당 하이라이트 (최대 스파이크 식사, 가장 안정적이었던 식사)
  4) **교차 상관 인사이트** (핵심): "일기에 '피곤/야근' 단어가 등장한 날, 저녁 식사 혈당 피크가 평균 대비 +23mg/dL 높았어요" 류의 개인 패턴 3~5개
  5) 다음 3일 실천 제안 3개 (구체적·측정가능)
  6) 생리주기 phase 맞춤 코멘트
· 생성 파이프라인: 서버가 3일치 정형 데이터 + 일기 분석 결과를 구조화 JSON으로 만들어 LLM에 전달 → 리포트 JSON 수신 → 앱에서 카드형 렌더링 (상세는 SRS §7)
· 리포트는 영구 보관, 히스토리 열람 가능
· 무료 사용자: 첫 리포트 1회 무료 체험 → 이후 페이월

### F-08. 대시보드 & 저속노화 점수 — P0 (기존 유지 + 보강)
· 월간 저속노화 점수, 4축 달성률, 14일 달성 기록, 생리기/배란기 달력 (기존 UI 계승)
· 혈당 통계 위젯 추가: 평균 혈당, 스파이크 횟수, 목표 범위 내 시간(TIR%) — CGM 사용자 한정

### F-09. 구독 & 결제 — P0
· 인앱 구독 (iOS StoreKit / Android Play Billing): 월 9,900원 "벨라모나 플러스"
· 구독 혜택: 3일 AI 리포트 무제한, 혈당 예측 무제한, 리포트 히스토리, Excel 내보내기
· 무료 티어: 기록 기능 전체 무료, AI 리포트 1회 체험, 혈당 예측 일 3회
· 웹 결제는 v1에서 제외 (스토어 정책 리스크 회피)

### F-10. 친구 연동·치팅데이·레시피 — P1 (기존 기능 서버화)
· 친구 초대 코드, 달성률 비교, 치팅데이 세이브 칼로리 — 기존 UX 유지하되 서버 데이터로 전환
· 저당 레시피: 정적 콘텐츠 CMS화, 쿠팡파트너스 링크 삽입 지점 유지

### F-11. 알림 — P0 (최소셋)
· 리포트 발행 푸시, 기록 리마인더(식사/일기/수면), 생리주기 phase 전환 알림

### F-12. 데이터 관리 — P0 (법적 필수)
· 내 데이터 전체 내보내기(JSON/Excel), 계정 삭제(30일 유예 후 파기), 동의 내역 조회·철회

## 5. 성공 지표 (KPI)
· 활성화: 가입 후 72시간 내 "일기 1 + 식사 3 + 혈당 1" 달성률 ≥ 40%
· 첫 리포트 열람률 ≥ 70%, 리포트 → 구독 전환율 ≥ 8%
· D30 리텐션 ≥ 25%, 구독 월 이탈률 ≤ 8%
· AI 원가율: 구독 매출 대비 AI API 비용 ≤ 15%

## 7. 사용자 플로우 (핵심 3개)

### 7.1 신규 → 첫 리포트 (활성화 루프)
가입 → 온보딩 → "3일 챌린지" 안내(일기·식사·혈당 기록 미션) → 72시간 데이터 충족 → 첫 무료 리포트 푸시 → 열람 → 페이월 노출

### 7.2 식사 → 예측 → 실측 학습 루프
음식 사진/검색 → AI 인식 확인 → 예상 혈당 반응 표시 → (CGM/수동) 실측 유입 → 예측 vs 실측 비교 카드 → 개인 보정계수 업데이트

### 7.3 일기 → 감정-행동 연결
일기 저장 → 키워드 분석 → 다음 리포트에서 "이 단어가 나온 날의 내 행동 패턴" 인사이트로 회수

## 9. 의료·규제 가드레일 (모든 문구·기능에 우선 적용)

· 벨라모나는 **비의료 웰니스 서비스**로 포지셔닝한다. 질병의 진단·치료·예방을 표방하는 문구 금지.
· 금지 표현 예: "PCOS를 치료", "인슐린 저항성을 진단", "혈당을 정상화합니다"
· 허용 표현 예: "생활습관 기록을 돕습니다", "참고용 추정치", "패턴을 함께 살펴봐요"
· 혈당 예측·AI 리포트 모든 화면 하단 고정 고지: "본 정보는 의료적 진단·처방이 아니며 참고용입니다. 건강 이상 시 의료 전문가와 상담하세요."
· 저혈당(70mg/dL 미만)·고혈당(250mg/dL 초과) 감지 시: 수치 안내 + "의료 전문가 상담 권고" 안전 메시지 (치료 지시 금지)
· PCOS는 "관리 목적의 기록 도구" 맥락으로만 언급. 식약처 SaMD(소프트웨어 의료기기) 해당 여부는 제품 오너(변리사)가 별도 법률 검토 — 기능 변경으로 진단·치료 성격이 생기면 즉시 보고.
