작성일: 2026-07-07 | 채널: 인스타그램 릴스(메인) · 유튜브 쇼츠 · 틱톡 · 블로그(전환 허브)
사용 도구: Gemini Omni(구 Veo, Gemini 앱 내 영상 생성·편집) + Flow(장면 확장) + CapCut(최종 조립)

## 0. 이 문서 사용법 (Gemini에 넣는 방법)

1. 촬영 소스 준비: §7의 촬영 리스트대로 본인 폰으로 찍은 동영상 1개 + 사진(최대 5장)을 준비한다.
2. Gemini 앱 → 동영상 생성 → 사진/동영상 업로드 → §8의 해당 에피소드 프롬프트를 **씬 1부터 순서대로** 붙여넣는다. (Gemini Omni는 이미지 최대 5장 + 동영상 1개 참조, 멀티턴 편집 지원. 기본 클립은 약 8초 → 씬별로 뽑아 이어붙이거나 Flow의 '장면 확장'으로 연장해 1분을 만든다.)
3. 각 씬 생성 후 "이전 클립의 마지막 장면에서 자연스럽게 이어서" 지시로 다음 씬을 확장한다.
4. CapCut에서 자막 스타일(§5.4)·음원·앱 화면 녹화 삽입 → 워터마크 없는 원본으로 각 플랫폼에 개별 업로드(타 플랫폼 워터마크 있으면 노출 제한됨).
5. **필수**: AI 생성 장면이 포함된 사실적 영상은 인스타/유튜브의 'AI 생성 콘텐츠' 라벨을 켠다. 캐릭터 계정 소개에도 "AI 캐릭터"임을 명시한다(§9 참조 — 신뢰·법적 리스크 방지).

## 3. 2026 바이럴 포맷 분석 → 벨라모나 적용

현재 알고리즘이 보는 것: <cite index="18-1">단순 조회수가 아닌 체류 시간(Retention)과 재생산(Remix). 승부는 3초가 아니라 1초 안에 결정되며, 시작과 동시에 결과물·질문·시각적 충격을 던져야 하고, 끝과 처음이 이어지는 무한 루프 편집이 필수</cite>. 또한 <cite index="18-1">과하게 연출된 스튜디오 영상보다 폰으로 무심하게 찍은 듯한 날것(Raw) 질감을 선호</cite>하고, <cite index="12-1">첫 훅·세로형 1080×1920·자막은 기본값</cite>이다. <cite index="13-1">릴스는 감성·팬덤 커머스, 쇼츠는 정보·검색 재노출에 강하다</cite>.

벨라모나에 적용할 6대 포맷:
· ① 실험형 (최강): "같은 떡볶이, 먹는 순서만 바꿨더니?" → 혈당 그래프 비교가 곧 시각적 충격. CGM 그래프 = 우리만의 비주얼 자산
· ② 비포/애프터 1초 훅: 혈당 스파이크 그래프를 0초에 먼저 보여주고 시작
· ③ POV/브이로그: "다낭성 직장인의 점심시간 POV" — 날것 핸드헬드 질감
· ④ 반전형: "다이어트 콜라면 괜찮겠지? → 내 혈당: ㅋ"
· ⑤ 챌린지형: "혈당 스파이크 없는 편의점 한 끼 챌린지" (리믹스 유도)
· ⑥ 무한루프: 마지막 대사가 첫 대사로 이어지게 대본 설계 (모든 에피소드 공통 적용)

```

씬2: 침대에서 폰에 일기 쓰는 모나. 화면에 '야근', '피곤' 단어 강조 없이 타이핑.

씬4: 모나 현실 리액션 "소름. 감정이 혈당을 움직이고 있었어."
```

## 10. 1시간 롱폼 기획: 「혈당 지키는 1시간 — 같이 걷고 같이 기록해요」

포지션: 스터디윗미의 혈당 버전 = "워크윗미(Walk with me) + 식후 루틴 라이브". 유튜브 롱폼·인스타 라이브 겸용, 쇼츠 10개의 원천 소스.

구성 (60분):
· 00:00–03:00 오프닝: 오늘의 식사 공개 + 벨라모나 예측 화면 ("앱은 피크 155 예상. 맞나 보자")
· 03:00–08:00 거꾸로 식사법으로 식사 (ASMR 톤)
· 08:00–48:00 식후 40분 산책/홈 스텝 세션: 긍정 확언 멘트 + 잔잔 업비트 음악(저작권 프리), 10분마다 실시간 혈당 체크인 자막 ("30분 경과: 128, 순항 중 🦋")
· 48:00–55:00 그래프 리뷰: 예측 vs 실제 비교
· 55:00–60:00 3분 일기 타임(시청자 함께) + 클로징 "오늘 데이터 하나 벌었다"
멘트 톤: 명령 아닌 초대형("같이 가볼까요"), 자책 금지 언어. 챕터 타임스탬프 필수(검색 재노출).
재활용: 실시간 혈당 체크인 순간들만 잘라 쇼츠 E8 계열로 재편집.
